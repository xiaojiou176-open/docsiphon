# Support

## Scope

Docsiphon is maintained as an open-source CLI project.

Support is best-effort for:

- installation and local setup questions
- reproducible exporter bugs
- documentation drift in the tracked public docs surface
- security reports routed through `SECURITY.md`

Support is not a guarantee of vendor certification, hosted operations, or
site-specific scraping success across every documentation stack.

## Before Opening an Issue

Please gather:

1. the exact command you ran
2. the target URL or a sanitized equivalent
3. the output root you used
4. the verification commands you already ran
5. any relevant `manifest.jsonl`, `report.json`, or `_errors/` evidence

## Canonical Entry Points

- project overview and commands: `README.md`
- repository map and execution model: `docs/README.md`
- contribution workflow: `CONTRIBUTING.md`

## Response Expectations

There is no formal response SLA. Repository help depends on maintainer
availability.
